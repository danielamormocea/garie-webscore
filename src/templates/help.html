EEA websites are audited daily and receive a score between 0 and 100 for each of the following metrics:

  • Performance - evaluated using Google Chrome Developer Tools
  • Progressive Web App - evaluated using Google Chrome Developer Tools
  • Accessibility - evaluated using Google Chrome Developer Tools (Audits)
  • Best Practice - evaluated using Google Chrome Developer Tools (Audits)
  • SEO Score - evaluated using Google Chrome Developer Tools (Audits)
  • Links Integrity - the amount of broken links in websites is evaluated using linkchecker (https://github.com/linkchecker/linkchecker), which is run locally in a Docker container
  • Encryption (TLS) - certificates validity and configuration are evaluated using the tool at https://www.ssllabs.com/ssltest/
  • SecurityHeaders.com - Checks for missing HTTP security headers in website responses using the tool at https://securityheaders.com/. This is somewhat outdated and mostly superseded by the next metric.
  • Securityheaders(mozilla) - also checks for missing HTTP security headers in website responses, but uses the tool at https://observatory.mozilla.org/
  • Uptime (30 days) - this metrics gives 100 if the site is never found down/unresponsive within the checks performed under the last 30d period.
  • Server Errors per visit - aggregation of the number of server-side errors/exception received in Sentry (sentry.eea.europa.eu) from the website, divided by the total number of website visits as logged by Matomo (https://matomo.org/)
  • JS Errors per visit - aggregation of the number of JS errors/exceptions received in Sentry(sentry.eea.europa.eu), divided by the number of website visits as logged by Matomo (https://matomo.org/)
  • Test coverage - code quality metric that is computed based on the unit test coverage in Jenkins (ci.eionet.europa.eu)
  • Bugs - code quality metric that is computed based on the results of tests ran in Jenkins (ci.eionet.europa.eu) using the SonarQube plugin
  • Vulnerabilities - code quality metric that is computed based on the results of tests ran in Jenkins (ci.eionet.europa.eu) using the SonarQube plugin
  • Code smells - code quality metric is computed based on the results of tests ran in Jenkins (ci.eionet.europa.eu) using the SonarQube plugin
  • Duplication score - code quality metric is computed based on the results of tests ran in Jenkins (ci.eionet.europa.eu) using the SonarQube plugin
  • Privacy score - evaluated using https://privacyscore.org/
  • webbkoll.dataskydd.net - evaluated using Webbkoll (webbkoll.dataskydd.net), an online tool that simulates a normal browser visit and checks for potential privacy problems.

For more info please visit the quality metrics wiki page.
